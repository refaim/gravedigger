## Строки произвольной длины в EXE

Сейчас ExeTextHandler требует что замена строки <= оригинальной длины (pad нулями).
Для перевода на другие языки этого может не хватить.

Варианты реализации произвольной длины:
1. **Relocate strings** — перенести строки в свободное место EXE, пропатчить все ссылки (нужен дизассемблер для поиска xref)
2. **Расширить сегмент данных** — увеличить code image, сдвинуть строки, пропатчить ссылки
3. **Использовать padding между строками** — если после строки есть нули, занять их (ограниченно)

Сложность: код игры ссылается на строки по абсолютным адресам/смещениям, нужно найти все xref через disasm (x86 16-bit real mode).

## Русификация шрифта

Игра использует встроенный bitmap-шрифт (EGA). Для русского перевода нужно:
1. Найти шрифт в EXE — определить смещение, формат (bitmap glyphs), размер символа
2. Понять маппинг символов — скорее всего ASCII 0x20-0x7F, нужно добавить кириллицу
3. Варианты:
   - **Заменить неиспользуемые ASCII-символы** на кириллицу (ограниченно, может не хватить слотов)
   - **Расширить таблицу шрифта** — добавить glyphs для 0x80-0xFF (CP866 или свой маппинг), пропатчить код рендеринга
   - **Перерисовать шрифт** — нарисовать кириллические глифы в том же стиле/размере
4. Нужно исследовать: как игра рендерит текст (INT 10h? собственный рендерер?), есть ли таблица ширин (proportional vs monospace)
